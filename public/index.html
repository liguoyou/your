yarn 全局安装（yarn global add [package]）后，命令不生效

2020-05-29

背景

最近使用 npm/cnpm 很不爽, 尝试将自己的项目全部转成 yarn

问题描述

使用 yarn 全局安装 @vue/cli, hexo-cli 等, 安装成功, 但是使用命令却不生效…

yarn global add hexo-cli

解决过程

第一个感觉就是环境变量问题了, 本人是win10

(1) 找到 yarn 的全局安装目录 yarn global bin

(2) 将该目录地址拷贝至环境变量-系统变量-path 中:

(3) 重启终端 git bash, 执行命令, 应该就可以了.

好吧, 我的还不可以

卸载全局依赖包 yarn global remove hexo , 再重新安装 yarn global add hexo

还是不行

然后我发现我的安装文件在 C:\Users\guoyo\AppData\Local\Yarn\Data\global\node_modules\.bin 中,

尝试将刚刚的环境变量地址修改成这个, ok, 重启终端成功了

原因

  1. 可能是从 npm/cnpm 切换为 yarn 某个环节不对
  2. 可能是安装 yarn 的时候造成的

https://blog.csdn.net/weixin_41643133/article/details/83829235

基于 Hexo 搭建 Gitee 个人静态博客

2020-05-29

首次搭建博客

1. 全局安装 Hexo 这里我使用的是 yarn

yarn global add hexo-cli

或者

npm install -g hexo-cli

2. 初始化文件夹

创建 文件夹 hexo, 在 hexo 中打开 git Bash, 执行:

hexo init 网速不好的话, 这里可能需要点时间

init 后目录如下, 关于目录及文件的说明请查看官方文档(中文)

3. 编译&运行

打开代码, 从 package.json 文件中可以看到:

执行 yarn build 进行编译, 再执行 yarn server 可以运行服务

当然, 更便捷的操作是执行

hexo generate && hexo server

简写

hexo g && hexo s

4. 首次运行如图 默认主题 landscape

安装主题

1. 选择主题 我这里选择的主题是 Ocean,

在官方的主题里面选择一个你喜欢的主题, 按照主题的文档来就好

下载好慢….等等等…

2. 重启

hexo clean && hexo g && hexo s 新的主题:

部署博客到Gitee上

1. 准备工作

(1) 首先到 Gitee 注册账号并创建仓库, 例如: your, 并开启 Gitee pages 服务

(2) 配置 _config.yml

(3) 安装自动部署发布工具
yarn add hexo-deployer-git

发布博客

hexo clean && hexo g && hexo d

进入 Gitee Pages 点击更新按钮, 等待部署完成即可

值得注意的是

执行hexo d之后, 会将编译后的文件 push 至仓库中

如图:

但是, 我想把整个本地的仓库 push 上去怎么做呢

那就将编译好的整个仓库的内容提交(git push)上去, (更多关于 git 的操作)

然后设置部署目录为 public 即可
